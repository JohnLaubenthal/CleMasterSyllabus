The wonderful world of Loops

  • In prior lessons we covered while and do while and for loops.

  • We also discussed some of the situations in which we would use them.

  • Now we will discuss the foreach loop.

  • The foreach loop will be one of the most important tools C# has to offer.

  • We predominantly use a foreach loops when we are iterating through an array, list, dictionary, or other data structure.

Understanding the foreach loop structure

  • The foreach loop is one of the least error prone loops we will use in programming.

  • The structure of a foreach loop is differant from that of a for loop.

  • As with the for loop, the foreach keyword tells the compiler to execute a foreach loop.

  • Unlike a for loop, we do not use a index variable, .length property, or and increment/decrement statement.

  • One of the most common error’s Array out of bounds exception does not apply to foreach loops.

Foreach loop array example

  • Here is an example of a foreach loop. Lets walk through its parts.

  • You will notice a foreach loop is written in plain english.

int[] fibarray = new int[] { 0, 1, 1, 2, 3, 5, 8, 13 };
foreach (int element in fibarray)
{
    System.Console.WriteLine(element);
}

  • The foreach loop operates just as it is written.

  • The loop will execute for each int element in the fibarray.

  • We do not have to keep track of the length, nor do we have to increment with each iteration.

Foreach loop list example

  • Here is another example of a foreach loop.
List<int> fibarray = new List<int> { 0, 1, 1, 2, 3, 5, 8, 13 };
foreach (int element in fibarray)
{
    System.Console.WriteLine(element);
}

  • The foreach loop operates just as it is written.

  • The loop will execute for each int element in the fibarray.

  • We do not have to keep track of the length, nor do we have to increment with each iteration.

The body

  • With the exception of list operations such as .Add(), .Remove(), .Insert(), etc the body is the same as a for loop.

Summary

  • The for loop and the foreach loop perform the same functions.

  • Although their inital set-up statements are differant, the work done in the body is the same.
